# Deliverables

Non-code work products live here: research reports, written content, data outputs/datasets, analyses,
decks, and other documents the work produces (work_type: document | data | research | ops).

```text
deliverables/
├── research/    investigations, comparisons, findings (sources cited — R-024-04)
├── content/     written/creative outputs
└── data/        produced datasets/exports (synthetic or licensed only — R-013)
```

Rules of the room: every deliverable traces to a purpose_ref (R-024), claims cite sources, and QC
verifies the acceptance criteria for its work_type just like it does for code (R-008/R-022-08).
Register deliverable roots in `.maestro/registry/components.yaml` when agents need scoped write access.

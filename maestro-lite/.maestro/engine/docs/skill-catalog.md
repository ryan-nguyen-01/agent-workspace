### Workflow (control-plane skills) (12)

- **`skill-agent-factory`** — Generate component-specific coder agents from project brain and approved component scopes.
- **`skill-bug-routing`** — Classify QC defects as blocker or non-blocker and route fix work to the correct generated coder through Coder Leader.
- **`skill-coder-leader`** — Coordinate generated service coder agents for single-service and multi-service implementation tasks.
- **`skill-dev-verification`** — Decide Code Done using critical checks, component test policy, scope compliance, and >=80% dev verification score.
- **`skill-memory-update`** — Persist durable project, service, task, agent, and QC learnings after meaningful workflow events.
- **`skill-project-brain`** — Manage reusable project brain files so agents can resume work across conversations without rescanning the whole repository.
- **`skill-project-onboarding`** — Build the initial project brain by scanning architecture, services, stack, dependencies, environment, and test policy.
- **`skill-qc-handoff`** — Create the mandatory Dev-to-QC handoff document after Code Done.
- **`skill-qc-runner`** — Run QC from the handoff document, record test results, and stop immediately on blocker bugs.
- **`skill-service-coder`** — Implement code inside a generated service coder's allowed scope while following component test policy and handoff obligations.
- **`skill-task-analysis`** — Convert HLD, LLD, ticket text, bug reports, or direct user text into a normalized task spec before implementation.
- **`skill-workflow-policy`** — Validate workflow state transitions, required artifacts, approval gates, and agent permissions.

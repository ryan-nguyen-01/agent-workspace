# Platform Changes & Migration (Flutter 3.32–3.41)

## UIScene Lifecycle Migration (iOS — critical)

Apple mandates UIScene lifecycle for UIKit apps in the iOS release after iOS 26. All existing Flutter iOS apps must migrate.

```bash
# Automatic migration (experimental in 3.38, default in 3.41)
flutter config --enable-uiscene-migration
```

Or follow manual migration guide. Plugins using app lifecycle events must also update.

**Timeline:**
- Flutter 3.38: Opt-in via `flutter config --enable-uiscene-migration`
- Flutter 3.41: UIScene lifecycle is the default

## Breaking Changes by Version

### Flutter 3.32 / Dart 3.8

- `ExpansionTileController` deprecated → use `ExpansibleController`
- `ThemeData.indicatorColor` deprecated → use `TabBarThemeData.indicatorColor`
- `CardTheme`/`DialogTheme`/`TabBarTheme` types → migrate to `CardThemeData`/`DialogThemeData`/`TabBarThemeData`
- `firebase_vertexai` → migrate to `firebase_ai` (supports both Gemini API and Vertex AI)
- `flutter_markdown`, `flutter_adaptive_scaffold`, and 4 other first-party packages discontinued

### Flutter 3.35 / Dart 3.9

- `Radio.groupValue` and `Radio.onChanged` deprecated → use `RadioGroup` widget
- `DropdownButtonFormField.value` deprecated → renamed to `initialValue`
- `Form` can no longer be used directly as a sliver — wrap in `SliverToBoxAdapter`
- Minimum Android SDK → API 24; requires Gradle 8.7.0, AGP 8.6.0, Java 17
- `pluginClass: none` in plugin pubspec deprecated — remove the field entirely
- `NavigationRail` now supports scrolling; `NavigationDrawer` gains `header`/`footer` params
- `fullscreenDialog` property added to `ModalRoute` and `showDialog`
- `FormField` gains `onReset` callback

### Flutter 3.38 / Dart 3.10

- Default Android page transition → `FadeForwardsPageTransitionsBuilder` (was `ZoomPageTransitionsBuilder`); predictive back enabled by default in `MaterialApp`
- `OverlayPortal.targetsRootOverlay` deprecated → use `OverlayPortal(overlayLocation: OverlayChildLocation.rootOverlay)`
- `SnackBar` with an action no longer auto-dismisses
- `version` file at Flutter SDK root removed → use `bin/cache/flutter.version.json`
- `AssetManifest.json` no longer generated by default
- Flutter 3.38 requires Java 17 minimum for Android
- Android: NDK r28 default (required for 16KB page size support)

### Flutter 3.41 / Dart 3.11

- UIScene lifecycle is now the default for iOS (was opt-in in 3.38)
- New plugin projects default to Kotlin DSL for Gradle
- **Do not** upgrade to AGP 9 yet — plugin migration not supported

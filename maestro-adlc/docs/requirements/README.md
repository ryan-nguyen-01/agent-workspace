# Requirements

Store capabilities, features, user stories, use cases, business rules, and non-functional requirements
here. Every implementation task must reference an approved requirement or explicitly record why none exists.

Follow the **BA Documentation Standard**: `.maestro/engine/docs/ba-documentation-standard.md` — BRD/PRD/use-case/user-story/business-rules/NFR/RTM templates, IDs, and which are required for MVP vs production. Store docs in the matching subfolder; every task must trace to an approved requirement (RTM).

# Agent Catalog

`agent-workspace` has **4 agent classes**. Keep these terms distinct in docs, registry, handoff.
See the full classification in [`.agent/docs/agent-taxonomy.md`](../../.agent/docs/agent-taxonomy.md).

```text
.claude/agents/
├── workflow/      12 control-plane agents (routing, gates, state machine)
├── coders/        built-in + generated service coders (write code, scoped)
└── specialists/   19 specialist advisors (advise in-pipeline, do NOT code)
```

| Class | Count | Role | Decides gates? | Writes app code? |
|-----|----------|---------|------------------|----------------|
| Workflow agents | 12 | Pipeline routing, approval gates, state transitions, verification, QC, memory | ✅ | ❌ |
| Built-in coders | 2 | `coder-infra`, `coder-database` — cross-cutting code | ❌ | ✅ (scoped) |
| Generated coders | n | Service-specific, generated by `agent-factory` after onboarding | ❌ | ✅ (scoped) |
| Specialist advisors | 19 | Domain experts, evidence-based advice | ❌ (recommend only) | ❌ |

> **Principle:** Everything goes through `coordinator` (single entrypoint). Specialist advisors are **not** standalone entrypoints — a workflow agent invokes them in-pipeline.

## 1. Workflow agents → [`workflow/`](workflow/)

| When you need… | Use agent | Model |
|----------|-----------|-------|
| Route/coordinate every task, enforce gates | `coordinator` | fast_router |
| Scan project, create the project brain | `onboarding` | deep_reasoning |
| Create service coder agents | `agent-factory` | coding_planner |
| Normalize a task into spec + context_plan | `task-analysis` | deep_reasoning |
| Review architecture/contract/risk | `solution-architect` | deep_reasoning |
| Plan + assign coders + review code | `coder-leader` | coding_planner |
| Evaluate Code Done (≥80% + critical checks) | `dev-verification` | verification |
| Create the Dev→QC handoff | `qc-handoff` | fast_router |
| Run QC tests, stop on a blocker | `qc-runner` | verification |
| Classify a bug blocker/non-blocker | `bug-router` | deep_reasoning |
| Persist durable learnings | `memory-update` | memory_light |
| Validate transitions/gates/artifacts | `workflow-policy` | deep_reasoning |

## 2. Coders → [`coders/`](coders/)

| When you need… | Use agent | Origin |
|----------|-----------|--------|
| Terraform/IaC, K8s, Docker, CI/CD | `coder-infra` | built-in |
| Schema, migrations, queries, indexes | `coder-database` | built-in |
| Code one specific service | `coder-<service>` | generated (agent-factory) |

## 3. Specialist advisors → [`specialists/`](specialists/)

See the full quick-selection table in [`specialists/README.md`](specialists/README.md). Summary by category:

- **architecture/** — api-designer, database-architect, cloud-architect, event-architect, ui-ux-designer
- **quality-security/** — security-auditor, performance-engineer, accessibility-auditor, code-reviewer
- **product/** — discovery-analyst, business-analyst, product-strategist
- **data-ai/** — data-engineer, ml-ai-architect
- **ops-devex/** — sre-observability, technical-writer, migration-strategist
- **research-qa/** — tech-researcher, qa-strategist
